# 统计功能

## 功能描述

先筛选统计类型，再根据统计类型提供具体选项。

可将统计选项保存为方案，后续可选择方案直接进行统计分析。

方案中的时间记录为相对于“基准时间”的时间差。“基准时间”的定义如下

```typescript
function getBaseTime() {
    const baseTime = new Date()
    baseTime.setHours(0, 0, 0, 0)
    return baseTime
}
```

随后每次选中方案时，根据方案中的时间差，计算出相对于“基准时间”的时间。

下文中“多选对比”是指用户可以添加多个该选项以进行对比分析。

### 按应用统计的统计类型

#### 应用使用总时长

- 筛选时间范围（时间和日期，必选）
- 筛选应用分类（单选，可选）
- 展示样式（卡片、进度条、饼图{小于5%记为其他}，必选）

#### 应用使用天数

- 筛选时间范围（日期）
- 筛选应用分类（单选，可选）
- 展示样式（卡片、饼图{小于5%记为其他}，必选）

### 按分类统计的统计类型

#### 分类使用总时长

- 筛选时间范围（时间和日期，必选）
- 筛选应用分类（多选，可选）
- 展示样式（卡片、进度条、饼图{小于5%记为其他，若未选择分类，则展示所有分类，并层叠展示所有分类}，必选）

#### 分类使用天数

- 筛选时间范围（日期）
- 筛选应用分类（多选，可选）
- 展示样式（卡片、饼图{小于5%记为其他}，必选）

#### 分类使用节律（每天、每周的使用时间分布）

- 选择时间跨度（天、周，必选）
- 选择统计粒度（在时间跨度内每单位的时间，必须能被时间跨度整除{例如时间跨度为1天，统计粒度为6小时、4小时、2小时、1小时、30分钟、15分钟、5分钟等}）
- 筛选时间范围（日期{必须是整日或整周}，必选，多选对比）
- 筛选应用分类（多选对比，可选，不选则不区分分类）
- 展示样式（极坐标、柱状图，必选）

## 实现细节

- 传入时间参数均为时间戳
- 时间范围为[开始时间, 结束时间)
- 时间校验合法
  - 若包含跨度，例如按天统计则时间范围必须是整日
  - 若包含粒度，则粒度一定能整除时间跨度且时间跨度相较于时间粒度的倍数不能超过 24 * 60
- 当允许多选时间范围时，提供多个选择器来选择
- 当允许多选应用分类时，为单个选择器允许多选
