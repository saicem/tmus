# 分类功能

用户可以将应用进行分类，分类是层级的，每个分类下面可以有多个子级分类，同一父级的分类名称不能相同。应用只能属于一个分类。

## 可用操作

操作若涉及具体的分类或应用均以id为参数。

- 添加分类
- 更新分类
- 删除分类
- 为应用分配分类，需注意如果应用已经被分配了分类需要删除旧分类
- 移除应用的分类（删除父级会同时删除子级）
- 查询未被分类的应用（限制查询范围，使用 offset 和 limit 参数。可通过关键词搜索，名称、公司、路径任意包含关键词则返回）
- 查询某分类下的应用（返回分类下的应用详情）
- 查询完整的分类树（返回分类树以及分类下的应用id）

## 实现细节

分类的修改先应用到内存中，由定时任务写入文件。
启动时从文件读取分类信息到内存，之后都是从内存中读取分类信息。
需要建立分类id到分类详情的映射，以及应用id到分类id的映射。

## 数据结构

示例以 TypeScript 定义数据结构。实际根据编程语言决定具体定义。

```typescript
interface Category {
    id: string;
    parent_id: string | null;
    name: string;
    children: Category[];
    app_ids: number[];
}
```
